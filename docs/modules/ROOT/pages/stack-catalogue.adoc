= Stack Catalogue
:page-layout: default

Stacks combine a devcontainer, VS Code profile, agents, toolsets, and bundles into installable “Agent Packs” for specific personas. Profiles are exported from VS Code; devcontainers keep tooling reproducible; agents and toolsets are wired via JSON configs and prompts in this repository.

fullstack-js-ts::
* Persona: Full-stack JS/TS.
* What you get: Node/TypeScript devcontainer; agents: refactor-surgeon, test-writer, doc-surgeon; toolsets: local-dev, review-only.
* Install: run the OS script (e.g. `scripts/install/fullstack-js-ts-macos.sh`), then import the `.code-profile` in VS Code.

python-data-analytics::
* Persona: Data/ML engineers.
* What you get: Python devcontainer; agents: data-explorer, pipeline-refactorer, doc-surgeon; toolsets: local-dev, review-only.
* Install: run the OS script (e.g. `scripts/install/python-data-analytics-macos.sh`), then import the `.code-profile` in VS Code.

infra-ops-sre::
* Persona: Infra/Platform/SRE.
* What you get: Infra-focused devcontainer; agents: infra-reviewer, incident-scribe, doc-surgeon; toolsets: review-only.
* Install: run the OS script (e.g. `scripts/install/infra-ops-sre-macos.sh`), then import the `.code-profile` in VS Code.

Notes:

- Profile placeholders (e.g. `vscode/profiles-dist/fullstack-js-ts.code-profile`) exist for wiring; overwrite them with real VS Code exports via “Export Profile…” when using a stack seriously.
- Installer scripts are non-destructive scaffolds; they validate configs and point to devcontainer/profile paths. Profile import remains a manual VS Code step.
